# MA2BT

## [简体中文](README.md) | [English](README_EN.md) | [日本語](README_JP.md)

> **Modular Avatar** to **BlendTree** optimizer for **VRChat Avatars**  
> Having issues? Please report them via [GitHub Issues](../../issues). I will do my best to address and fix them.

MA2BT runs at the end of the Modular Avatar build process. It merges eligible Animator layers into a single Direct BlendTree, reducing the number of FX layers and improving avatar performance.

Booth: https://puddingkc.booth.pm/items/8309096

## How It Works

Modular Avatar generates an independent Animator layer for every responsive property (Object Toggle, Material Setter, Shape Changer, etc.), which adds runtime overhead.  
MA2BT analyzes these generated layers after the build and converts simple structures into BlendTree nodes within a shared layer.

```
Before Optimization:
MA Responsive: Hat       (Layer)
MA Responsive: Glasses   (Layer)
MA Responsive: Jacket    (Layer)
MA Responsive: Shoes     (Layer)

After Optimization:
MA_To_BlendTree_Layer    (1 Layer, 1 Direct BlendTree)
  ├── hat_param     → 1D BlendTree
  ├── glasses_param → 1D BlendTree
  ├── jacket_param  → 1D BlendTree
  └── shoes_param   → 1D BlendTree
```

Layers that cannot be safely converted (e.g., multi-parameter AND conditions, non-instant transitions) will remain unchanged.

## Usage

1. Select your **Avatar Root** object.
2. Add component: **Add Component > MA2BT > MA2BT**.
3. Build your avatar normally; MA2BT will run automatically during the optimization phase.
4. Check the `[MA2BT]` logs in the Console for conversion results.

> ※ Note: If you have AAO or other layer-merging plugins installed, the generated `MA_To_BlendTree_Layer` may be further merged by them. You can temporarily remove other optimizers to test the conversion effectiveness.

## Options

| Option | Default | Description |
|---|---|---|
| **Compact Mode** | ON | Only generates BlendTree thresholds for values with actual animations, reducing empty animations. |
| **Multi-State Layers** | OFF | Attempts to convert layers with multiple conditional states (e.g., multi-value int parameters). Off by default for safety. |
| **Scan All Layers** | OFF | Scans all matching FX layers, not just those generated by MA. |

## Conversion Criteria

A layer will be converted if and only if **all** the following conditions are met:

- Layer name starts with `MA Responsive:` (unless Scan All Layers is enabled).
- State machine contains 2 states: 1 default + 1 conditional (more supported if Multi-State Layers is enabled).
- All Entry Transition conditions use only a **single parameter**.
- All Transitions are instant (Duration = 0, no Exit Time).

## Common Skip Reasons

The following will NOT be converted:

- **Multi-parameter AND conditions** — e.g., relying on both a menu parameter and a parent state (`__ActiveSelfProxy`).
- **Non-instant transitions** — contains blend duration.
- **Complex structures** — contains Sub-State Machines or AnyState transitions.
